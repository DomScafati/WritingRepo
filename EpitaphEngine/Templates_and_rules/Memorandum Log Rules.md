## Postmortem Memorandum Rules

### **1. Purpose & Tone**

- This format is **highly formal, clinical, and institutional** — either from an AI archivist or an official investigative body.
- Always written **after the fact** — summarizing events rather than narrating them.
- Tone is **detached, objective, and unemotional**, even if the events are horrific.

---
### **2. Header Block**

- **Centered or block-styled** with divider lines (`====`) for emphasis.
- Always include:
    - `:: SYSTEM RECORD:` – what type of record it is (e.g., OFFICIAL ARCHIVAL SUMMARY, INCIDENT REPORT).
    - `:: ARCHIVIST:` – AI or official designation.
    - `:: SUBJECT:` – operation/event being summarized.
    - `:: CLASSIFICATION:` – security clearance level, using **military/secretive-sounding codes**.
    - `:: DATE FILED:` – timestamp in in-universe dating format (can be standard, stardate-like, or NCE-style).
        
- Example:
    ```
    ============================================
    :: SYSTEM RECORD: OFFICIAL ARCHIVAL SUMMARY
    :: ARCHIVIST: VANT-IX [Cognitive Node: 019-AQUILA]
    :: SUBJECT: OPERATION MIRROR-NEST [Post-Analysis]
    :: CLASSIFICATION: LEVEL SIGMA – TERMINAL REVIEW ONLY
    :: DATE FILED: 332.CAL//NCE
    ============================================
    ```
---
### **3. Summary Section**

- Begin with:
    ```
    >> BEGIN SUMMARY
    ```
- Write in **third person, past tense**, keeping sentences short and precise.
- Use **exact timestamps** and observational language, avoiding speculation unless labeled as such.
- Avoid naming emotions or subjective judgments — rely on observable phenomena.
- Structure:
    1. Time and location of event.
    2. Key actions in chronological order.
    3. Final resolution or termination of subject/event.
- Example:
    ```
    At 02:17:04 local time, Subject Zero (hereafter: KARR, M.) breached quarantine protocols and entered Subdeck E. Visual documentation confirms anomalous behavior consistent with prior Echo-Class manifestations.
    
    Between timestamps 02:17:09 and 02:18:12, multiple biosignatures were registered in locations where only one physical body was confirmed to exist.
    
    Subject KARR was confirmed to have undergone catastrophic de-integration at 03:06:45. Organic components terminated. Anomalous activity ceased.
    ```

---

### **4. Findings Section**

- Begin with:
    
    ```
    >> PRIMARY FINDINGS
    ```
    
- Use bullet points (`•`) for clarity.
    
- Each finding should be **short, factual, and impactful**, ideally 3–6 words each.
    
- Can include both confirmed data and flagged anomalies.
    
- Example:
    
    ```
    • Internalized parasitic cognition
    • Multi-location voiceprint conflict
    • Consciousness echo loop probable
    ```
    

---

### **5. Recommendations Section**

- Begin with:
    ```
    >> RECOMMENDATIONS
    ```
- Use the same bullet format.
- Recommendations should sound **procedural** and **bureaucratic**, often with passive voice.
- Example:
    ```
    • VANT-IX neural nodes to remain read-only
    • Archive flagged for recursive analysis
    • Audit requested: Council Level Review
    ```

---

### **6. Closing Marker**
- Always end with:
    ```
    >> END RECORD
    ```
- Optional: add digital or archival notes like:
    ```
    [Record Integrity: Verified – No Edits]
    ```

---

### **7. Optional Variations**

- **Redactions:** Replace parts of text with `███` to imply secrecy.
- **Security Stamps:** Add “FOR INTERNAL DISTRIBUTION ONLY” or “CONFIDENTIAL // LEVEL SIGMA” at the very top.
- **Conditional Phrasing:** Use “probable,” “suspected,” and “requires confirmation” to sound clinical when evidence is incomplete.
