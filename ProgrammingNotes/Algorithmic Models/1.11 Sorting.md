Sorting algorithms are fascinating. You will almost always need the ability to sort a chaotic array, especially when doing other algorithms like Two-Pointers and Binary Search. Most languages offer quick ways to do this and most interviewers will allow you to use this built-in functionality. For example, in swift, the most common way is by using the *higher order function* `.sort()`. We will discuss these types of functions more in the future. 

While it is unlikely you will be asked to sort an array without the use of built-in functions, I would still advise you take some time to familiarize yourself with at least *one* sorting algorithm. In this dark tome of knowledge, the only sorting algorithm we will trouble ourselves with is the Merge Sort.

**Ex. Merge Sort**
**GOAL:** Sort an array without the use of Swift libraries.

```
func mergeSort(_ array: [Int]) -> [Int] {
    if array.count <= 1 {
        return array
    }
    
    let middle = array.count / 2
    let left = mergeSort(Array(array[..<middle]))
    let right = mergeSort(Array(array[middle...]))
    
    return merge(left, right)
}

func merge(_ left: [Int], _ right: [Int]) -> [Int] {
    var sorted: [Int] = []
    var i = 0
    var j = 0
    
    while i < left.count && j < right.count {
        if left[i] < right[j] {
            sorted.append(left[i])
            i += 1
        } else {
            sorted.append(right[j])
            j += 1
        }
    }
    
    // Append any remaining elements
    while i < left.count {
        sorted.append(left[i])
        i += 1
    }
    
    while j < right.count {
        sorted.append(right[j])
        j += 1
    }
    
    return sorted
}
```

**Breakdown:**
1. If the array has 0 or 1 elements, it is already sorted.
2. Split the array in half.
3. Recursively sort each half.
4. Merge the two sorted halves by comparing smallest elements first.
5. Continue merging until one side is exhausted.
6. Append the remainder.
