There are many layers to the dark arts of *algorithmic programming*. Think of any given coding challenge like an adversary from the Realm of Light. These planar constructs are sent from above to stop any worthy Fiend from roaming the Earth. Make no mistake, coding challenges are no different. They are designed specifically to test you (in a way that you will likely never use practically) and remove you from the running of that well-paying job.

*Coding challenges* like all warriors, possess a weakness, an Achilles heel that can be exploited to slay them. Before we continue our journey through this codex, it is critical to familiarize yourself with these sorts of coding challenges, which can be done by focusing on specific algorithms.

In this section of the tome, we will focus on the eight most vital algorithms Interviewers will use to test your mettle. If you find you are adept at these core tools, proceed in your own studies to learn deeper, darker arts.

### Cheat Sheet

| Pattern          | Description                                           | Typical Use Case                             | Complexity       |
| ---------------- | ----------------------------------------------------- | -------------------------------------------- | ---------------- |
| Single Pointer   | Walk through a list from start to finish              | Find max/min, sum, filter                    | O(n)             |
| Hash Map         | Store quick lookups using key/value pairs             | 2-sum (unsorted), duplicates, frequencies    | O(n) (avg)       |
| Two Pointers     | Use two indices moving at different speeds/directions | Palindromes, sorted 2-sum, cycle detection   | O(n)             |
| Binary Search    | Repeatedly cut the search space in half               | Sorted search, first bad version             | O(log n)         |
| Stack            | Keep track of previous items in LIFO order            | Valid parentheses, monotonic stack, undo     | O(n)             |
| Matrix Traversal | Iterate through 2D grid via rows/cols or direction    | Rotate matrix, island counting, grid search  | O(m × n)         |
| Linked List      | Sequential node-based structure with pointer links    | Reverse list, detect cycle, merge lists      | O(n)             |
| Binary Tree      | Hierarchical node structure with left/right children  | DFS, BFS, height, balanced checks            | O(n) (traversal) |
| Sorting          | Arrange elements in order                             | Enable binary search, two pointers, grouping | O(n log n)       |

### What's Special About Each?

- **Single Pointer**: The foundation. Linear traversal. No tricks, just fundamental iteration.
- **Two Pointers**: Eliminates redundant comparisons by exploiting order. Often converts O(n²) brute force into O(n).
- **Binary Search**: Destroys half the search space per step. Logarithmic time is a different class of efficiency.
- **Hash Map**: Trades memory for speed. Converts nested loops into constant-time lookups.
- **Stack**: Preserves order-dependent state. Essential for problems involving symmetry, nesting, or monotonic constraints.
- **Matrix Traversal**: Forces awareness of boundaries, direction, and multi-dimensional iteration.
- **Linked List**: Tests pointer manipulation and edge case discipline (head, tail, null).
- **Binary Tree**: Recursion playground. Traversal patterns (DFS/BFS) unlock most tree problems.
- **Sorting:** This enables the use of other algorithms as well converting messy grouping problems into linear traversal.
  
### Tips for Spotting Patterns

Train your frail, mortal mind to recognize the language used in these coding challenges. Once you can do that, you will be able to narrow down which algorithm to utilize.

| If the Problem Says...                               | Pattern Likely To Help |
| ---------------------------------------------------- | ---------------------- |
| “Iterate through” / “Compute total” / “Find max/min” | Single Pointer         |
| "Organize an unsorted collection"                    | Sorting                |
| “Sorted” + “Find pair”                               | Two Pointers           |
| “Sorted” + “Search”                                  | Binary Search          |
| “Any duplicates?” / “Count frequency”                | Hash Map               |
| “Valid parentheses” / “Undo” / “Next greater”        | Stack                  |
| “Grid” / “2D array” / “Islands”                      | Matrix Traversal       |
| “Reverse list” / “Detect cycle”                      | Linked List            |
| “Traverse tree” / “Height” / “Levels”                | Binary Tree            |
