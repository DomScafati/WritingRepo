 What in the 9 layers of Hell am I speaking about when I say *O(n)*? This is what is called *"Big-O"* notation. This is often used in algorithmic programming to describe an upper bound growth rate. While it is *uncommon* you will have to strictly adhere to a time or space complexity, familiarizing yourself will prove to be a boon.

Where `n =` input size, below are some common complexities.

### Cheat Sheet

| Big-O        | Name         | Example Pattern    | Growth                         |
| ------------ | ------------ | ------------------ | ------------------------------ |
| `O(1)`       | Constant     | Access array index | Does not grow                  |
| `O(n)`       | Linear       | Single loop        | Grows proportionally           |
| `O(n²)`      | Quadratic    | Nested loops       | Grows very fast                |
| `O(log n)`   | Logarithmic  | Binary search      | Work is cut in half each time  |
| `O(n log n)` | Linearithmic | Merge sort         | Slightly faster than quadratic |
| `O(2ⁿ)`      | Exponential  | Recursive subsets  | Explodes rapidly               |

### Time Complexity

**Ex 1: Constant Time O(1)**
No matter how large the array is, this takes the same time.

```
func firstElement(_ arr: [Int]) -> Int? {
    return arr.first
}
```

**Ex 2: Linear O(n)**
If array doubles in size → runtime doubles.

```
func sum(_ arr: [Int]) -> Int {
    var total = 0
    for num in arr {
        total += num
    }
    return total
}
```

**Ex 3: Quadratic O(n^2)**
Growth is quadratic, a squaring effect. If `n = 100`, the entire function will run 10,000 times.

```
func printPairs(_ arr: [Int]) {
    for i in arr {
        for j in arr {
            print(i, j)
        }
    }
}
```


### Space Complexity

This type of complexity measures memory space being used, ie., creating more variables relative to input size `n`.

**Ex 1: Constant Space 0(1)**
Though the below code is *linear* time complexity, ie., *single loop O(n)*, it does *not* create a second collection. This means the space complexity is *constant O(1)*; the memory usage does *not* grow with `n`.

```
func sum(_ arr: [Int]) -> Int {
    var total = 0
    for num in arr {
        total += num
    }
    return total
}
```

**Ex 2: Linear Space 0(n)**
Space grows directly with `n`, making both *time* and *space* complexity *O(n)* here.

```
func duplicate(_ arr: [Int]) -> [Int] {
    var result: [Int] = []
    for num in arr {
        result.append(num)
    }
    return result
}
```
